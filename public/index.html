API Basejump: URL Shortener Microservice

User stories:
    1) I can pass a URL as a parameter and I will receive a shortened URL in the JSON response.
    2) If I pass an invalid URL that doesn't follow the valid http://www.example.com format, the JSON response will contain an error instead.
    3) When I visit that shortened URL, it will redirect me to my original link.
    4) If the long URL already exists in database, returns the shortURL already in database and does not create multiple same entries.**

Example creation usage:

https://rebot-fcc-third.herokuapp.com/new/http://foo.com:80

Example creation output:

{ "original_url":"http://foo.com:80", "short_url":"https://rebot-fcc-third.herokuapp.com/8170" }

Usage:

https://rebot-fcc-third.herokuapp.com/8170

Will redirect to:

http://foo.com:80
